What is a GSI?

A GSI is a generic system image which is basically a file with .img extenstion. It contains all system apps and libs. A GSI doesn't need any kind of zips needs to be flashed after it. If your device is Treble-enabled, all you have to do is flash the system image.

What is a semiGSI ?

A semiGSI comes with zip files which are flashed after flashing the system image. In this case the system image is not modified (but can be debloated) and the post GSI zips contain files which help in booting. These post zip files patch the ramdisk of the boot image which is basically the addition of some ramdisk files from the OEM ROM's boot image.

How ErfanGSIs are different from PhhGSIs ?

Most of the GSIs which Erfan provides are ported from closed source OEM systems like OnePlus, MIUI, ZenUI etc. They are based on Android but are not open source. These manufacturers modify the AOSP code according to their requirements and add various kind of features and all extra things which are closed source and therefore it is up to devs like Erfan to solve issues. That's the reason these GSIs can not be bug-free on every phone out there. In case of PhhGSI , its simply based on pure AOSP and its source codes are open to all. He can do framework level patching if something is broken for a device and that is the reason they are supported on large number of devices with very small amounts of bugs.
